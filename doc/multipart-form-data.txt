Field name and boundaries convention for multipart/form-data request bodies.
This concerns photo uploads by users. Metadata is created client-side.

Update: we're allowing multiple images per request. As such, client will be 
required to send number of images in custom header field: n-images.


Content-Type: multipart/form-data; boundary=RMErTosHXEALgcbemszP

--RMErTosHXEALgcbemszP
Content-Disposition: form-data; name="Description"; 

<Description>
--RMErTosHXEALgcbemszP
Content-Disposition: form-data; name="UploadDate"; 

<UploadDate>
--RMErTosHXEALgcbemszP
Content-Disposition: file; name="UploadedImage1"; filename="example.jpg"
Content-Type: image/jpeg

<Binary image data goes here>
--RMErTosHXEALgcbemszP
Content-Disposition: file; name="UploadedImage2"; filename="example.jpg"
Content-Type: image/jpeg

<Binary image data goes here>
--RMErTosHXEALgcbemszP
...
...
--RMErTosHXEALgcbemszP
Content-Disposition: file; name="UploadedImageN"; filename="example.jpg"
Content-Type: image/jpeg

<Binary image data goes here>
--RMErTosHXEALgcbemszP--
